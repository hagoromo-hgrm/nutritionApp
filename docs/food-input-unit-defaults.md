# 食品入力単位の初期値

## 位置付け

この表は、MEXT食品を食事へ追加するときの入力開始値です。推奨摂取量や実物の計量値ではありません。栄養値の基準は可食部100g当たりのまま保持し、表の重量は `inputUnitConversions` でgへ換算します。

換算重量は、可食部の正味量を明記しているEatreatの資料を優先しました。味の素パーク、DELISH KITCHEN、食品ロス資料の廃棄部込み重量は、単位候補、値の範囲、個体差の確認に利用しています。MEXT食品名から生・ゆで・乾燥・缶詰等の状態を特定できない場合は適用しません。

## 初期設定する単位

特記がない既定入力値は1単位です。

| 対象 | 入力単位 | 1単位の可食重量 | 既定入力値 | 適用条件 |

| もち米の穀粒 | 合 | 155g | 1合 | 炊飯前 |
| 水稲・陸稲のめし | 杯 | 140g | 1杯 | 炊飯済み |
| ロールパン | 個 | 30g | 1個 | ロールパンそのもの |
| ゆでうどん | 玉 | 200g | 1玉 | ゆで |
| ゆでそば | 玉 | 160g | 1玉 | ゆで |
| そうめん・ひやむぎ | 束 | 50g | 2束 | 乾 |
| マカロニ・スパゲッティ | 人前 | 100g | 1人前 | 乾 |
| ぎょうざの皮 | 枚 | 6g | 1枚 | 皮そのもの |
| しゅうまいの皮 | 枚 | 3g | 1枚 | 皮そのもの |

分類全体へ一律の重量を設定せず、資料と対応する名称だけに適用します。資料間で差があるドレッシング、ごま等は初期設定しません。

## 初期設定しない代表例

- 厚さを識別できない食パン
- 切り方・厚さ・骨や殻の有無で重量が変わる肉・魚
- 販売規格で内容量が変わる生鮮品の袋・束・パック
- 調理前後で重量が変わる食品へ、別状態の重量しか確認できない場合
- 資料間の差が大きく、MEXT名から大きさを識別できない食品
