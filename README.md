# Nutrition PWA

iPhoneのホーム画面から使う個人用の栄養管理PWAです。食事履歴と食品マスターはIndexedDBに保存し、初回アクセス後は保存済みデータをオフラインで扱えます。

## 開発

```bash
npm install
npm run dev
```

検証:

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

## データと安全性

- 食品、食事記録、お気に入り、設定はDexie.js経由のIndexedDBに保存します。
- 食品の新規登録・編集は食品タブに集約し、今日の記録画面は食品選択と食事記録に専念します。
- 食品ごとに基準単位を保持します。初期データではご飯を合、卵を個、バナナを本として登録しています。単位変換は推測しません。
- 食事記録は登録時点の食品スナップショットを保持します。食品マスター削除後も履歴は残ります。
- JSON復元は形式を検証してから実行し、実行前に現在データの自動退避をダウンロードします。
- CSVは閲覧・分析用で、復元には使いません。列順は要件定義書FR-013に固定しています。
- 個人データを含むJSONをリポジトリへ置かないでください。

## 一般食品データ

`src/data/initialFoods.ts` には動作確認用の初期サンプルを同梱しています。確認済みの日本食品標準成分表データを配布用形式へ変換する場合は、元CSVのライセンス・出典・版を確認した上で次を実行します。

```bash
python3 scripts/convert_food_data.py input.csv data/foods.json \
  --source-version '確認済みの版' \
  --source-url '公式URL' \
  --acquired-date 2026-07-15
```

変換スクリプトは欠損値を`null`として保持し、出典メタデータをJSONに記録します。

## 実機確認

本番相当のHTTPS配信後、Safariで開いてホーム画面へ追加します。初回読み込み後に通信を切り、食品検索・食事登録・集計・バックアップ出力ができることを確認してください。カメラはHTTPSまたはlocalhostでのみ利用でき、権限拒否時はバーコード番号の手入力へ切り替わります。

外部商品APIは初期状態では無効です。コード上の接続先はOpen Food Facts API v3です。設定画面にAPI利用設定を追加する場合は、Open Food Factsの利用条件、ODbL、帰属表示、取得データの確認導線を合わせて確認してください。
